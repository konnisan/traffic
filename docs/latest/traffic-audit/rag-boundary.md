# RAG 与知识边界说明

## 一期知识分层

| 类型 | 处理方式 | 是否进入 RAG |
| --- | --- | --- |
| 车辆档案、交易记录、通行事件 | FastAPI 确定性程序 | 否 |
| 政策制度、规则、案例 | Dify Dataset 文本 RAG | 是 |
| 专业三元组 | 人工维护后再使用 | 可选 |
| 扫描件、表格 PDF、版式强文档 | 二期视觉检索 PoC | 否 |

## 不默认自动抽图谱

一期不从任意文件自动抽取图谱关系。专业关系数据应由业务人员或系统提供清晰三元组，并保留来源和审核状态。

## ColPali 边界

ColPali 适合扫描件、表格 PDF 和复杂版式文档的页面级检索，但会增加模型加载、显存/内存、多向量索引和部署复杂度。

因此一期不把 ColPali 放入主流程。后续只有当文本 RAG 在扫描制度文件上的召回率不足时，再独立建设 `visual_rag_poc`，并以命中页码和页面证据作为验收对象。
