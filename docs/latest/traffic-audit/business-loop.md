# 交通稽核业务闭环设计

## 案件状态

- `draft`：案件已创建，样例数据未上传完整。
- `ready`：车辆、交易、通行事件三类数据已上传。
- `queued`：分析任务已提交。
- `analyzing`：确定性分析执行中。
- `pending_review`：分析完成，等待人工复核。
- `confirmed`：管理员确认异常。
- `rejected`：管理员驳回结论。
- `analysis_failed`：分析失败，保留失败状态和任务日志。

## 证据项

每条异常证据必须包含：

- 异常类型和风险等级。
- 事实描述。
- 源文件、行号、原始 `record_id`。
- 规则依据。
- 置信说明。

## 路径还原

路径还原按事件时间、业务序号和记录 ID 稳定排序。默认以收费站和门架时间线展示，不做地图推断。

一期内置规则：

- 入口或出口缺失。
- 事件时序冲突。
- 路径不连续。
- 计费结果不一致。

## 人工复核

报告草稿中的正式结论默认是“未经人工确认”。只有管理员复核后，案件才能进入 `confirmed` 或 `rejected` 状态。
