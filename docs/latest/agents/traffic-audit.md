# 交通稽核智能体

交通稽核一期提供基于样例数据的逃费路径还原、异常证据整理、知识依据检索和报告草稿。所有异常由确定性规则产生，智能体只负责解释与知识检索，正式结论必须由管理员人工复核。

## 使用流程

1. 进入「交通稽核」，创建案件并填写疑似车辆和分析时间窗。
2. 分别上传 `vehicles`、`transactions` 和 `passages` 数据集。
3. 三类数据校验通过后启动分析，系统异步生成路径、证据和 Markdown 报告草稿。
4. 管理员检查源记录和规则依据后确认或驳回结论。
5. 在智能体页面选择「交通稽核智能体」，可以根据案件 ID 追问路径和证据。

样例文件位于 `test/fixtures/traffic_audit`，包含正常通行、缺少入口、时序冲突、路径断点和费用异常场景。

## 数据契约

所有文件支持 UTF-8 CSV、XLSX 或 XLSM。`record_id` 在单个数据集中唯一，重复记录保留第一次出现的数据；系统会保留文件名、行号和原始记录 ID 作为证据引用。

### vehicles

| 字段 | 说明 |
| --- | --- |
| `record_id` | 车辆记录 ID |
| `plate_number` | 车牌号，导入时转为大写并移除空格 |
| `plate_color` | 车牌颜色 |
| `vehicle_type` | 车辆类型 |

### transactions

| 字段 | 说明 |
| --- | --- |
| `record_id` | 交易记录 ID |
| `plate_number` | 车牌号 |
| `entry_station` / `exit_station` | 入口和出口收费站编码 |
| `entry_time` / `exit_time` | ISO 8601 时间 |
| `charged_amount` | 实收金额 |
| `expected_amount` | 规则应收金额 |

### passages

| 字段 | 说明 |
| --- | --- |
| `record_id` | 通行事件 ID |
| `plate_number` | 车牌号 |
| `event_time` | ISO 8601 事件时间 |
| `event_type` | `entry`、`gantry` 或 `exit` |
| `location_code` | 当前收费站或门架编码 |
| `sequence_no` | 业务事件序号 |
| `next_location_code` | 路网规则指定的下一节点，末节点留空 |

## 一期规则

- **入口/出口缺失**：通行链没有以入口开始或以出口结束。
- **事件时序冲突**：业务序号与事件时间不是同向递增。
- **路径不连续**：事件的 `next_location_code` 与下一条事件位置不一致。
- **计费不一致**：`charged_amount` 与 `expected_amount` 不相等。

规则命中只表示疑似异常。每项证据包含源文件、行号、记录 ID、规则依据和人工复核提示。

## 知识库配置

在知识库页面创建 LightRAG 知识库，建议命名为「交通稽核知识库」，并按政策制度、稽核规则、历史案例建立目录。将知识库添加到交通稽核智能体的配置后，智能体才能检索业务依据。

通行流水不应上传到 LightRAG 或全局知识图谱；流水由案件分析服务处理，知识库只保存制度、规则和案例材料。

## 权限与边界

- 普通用户只能创建和查看本部门案件。
- 管理员可以复核本部门案件，超级管理员可以跨部门查看和复核。
- 智能体案件工具只读，不能修改案件或提交复核结论。
- 一期不包含自动处罚、外部系统流转、地图路网、实时流处理或 DOCX/PDF 套版。
