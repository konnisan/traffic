# 交通稽核智能体

面向高速公路收费稽核场景的智能化工作台。系统在保留问答、知识库、知识图谱、任务中心、用户与权限等基础能力的前提下，新增交通稽核案件闭环、路径还原、异常证据、报告草稿和人工复核流程。

## 项目概述

本项目围绕“收费稽核事实判断可复现、证据可追溯、结论可复核”的目标设计。业务流水由确定性程序处理，LLM、Dify、RAG 和知识图谱只用于政策依据检索、辅助解释和报告草稿生成，不直接参与正式定案。

核心原则：

- 业务流水不进入 RAG 或知识图谱。
- 稽核事实由确定性规则生成。
- 每条异常必须关联源文件、行号、原始记录 ID 和规则依据。
- 报告输出默认为草稿，必须经过人工复核。
- 智能体能力用于辅助研判，不自动处罚、不自动流转外部系统。

## 功能模块

### 1. 交通稽核案件管理

- 支持创建案件，记录车牌、部门、分析时间窗、状态和创建人。
- 支持上传三类样例数据：车辆档案、收费交易、通行事件。
- 导入时完成字段校验、车牌标准化、时间标准化和 `record_id` 去重。
- 支持案件分析、任务进度、失败状态、人工确认和驳回。

### 2. 路径还原

- 根据入口、门架、出口事件还原车辆通行链。
- 按时间线展示通行节点。
- 提供站点/门架编码的示意地图。
- 每个节点保留源文件、行号和原始记录 ID。

### 3. 异常检测

一期内置四类确定性规则：

- 入口/出口缺失
- 事件时序冲突
- 路径不连续
- 计费结果不一致

每个规则命中后都会生成标准证据项，包含异常类型、风险等级、事实描述、源记录引用、规则依据和置信说明。

### 4. 案例分析

- 展示车辆摘要、通行节点数、异常证据数和综合评分。
- 汇总通行链完整性、异常类型和证据溯源情况。
- 预留相似案例关联入口，后续可接入案例库检索。

### 5. 稽核报告

- 自动生成 Markdown 报告草稿。
- 展示逃费类型分布和异常证据摘要。
- 支持报告下载。
- Dify Workflow 可作为可选报告润色和依据摘要组件，但输出始终是草稿。

### 6. 组件式工作流实验

新增 `工作流实验` 页面，用于验证类似 Haystack 的组件式编排思路。

实验页支持：

- 从组件库添加节点
- 调整节点顺序
- 删除节点
- 查看节点输入输出
- 查看配置预览
- 模拟运行流程
- 生成工作流 JSON 定义

内置组件类型包括：

- 样例文件解析
- 路径还原
- 异常规则检测
- 政策依据检索
- 图谱关系查询
- 智能体解释
- Dify 报告草稿

该页面目前为前端原型，不影响现有交通稽核主流程。

## 技术实现

### 后端

- FastAPI 提供案件、数据上传、分析任务、复核和报告下载接口。
- SQLAlchemy + PostgreSQL 保存案件、任务、证据、复核和工作流调用记录。
- Tasker 负责分析任务、Dify 工作流调用和任务状态管理。
- 交通稽核事实判断由 Python 确定性服务实现。

### 前端

- Vue 3 + Ant Design Vue 构建工作台界面。
- 交通稽核详情页拆分为独立业务面板：
  - 路径还原
  - 案例分析
  - 稽核报告生成
  - 异常证据
  - 人工复核
- 登录页和首页已替换为交通稽核业务语义。

### 知识能力

系统保留以下能力，并服务于交通稽核业务：

- 问答：政策解释、案例追问、报告说明。
- 知识库：政策制度、稽核规则、历史案例材料。
- 知识图谱：规则、异常类型、证据项、案例之间的结构关系。
- 任务中心：展示分析任务、工作流任务和后台任务状态。

## 技术栈

- 后端：FastAPI、Python 3.12、Pydantic、SQLAlchemy
- 前端：Vue 3、Ant Design Vue、Less
- 数据库：PostgreSQL
- 任务执行：Tasker
- 知识检索：RAG、知识库能力
- 结构关系：知识图谱
- 可选编排：Dify Workflow
- 部署：Docker Compose

## 前台使用流程

1. 登录系统。
2. 进入 `交通稽核`。
3. 新建案件，填写车牌号和分析时间窗。
4. 上传三类数据：
   - `vehicles.csv`
   - `transactions.csv`
   - `passages.csv`
5. 点击 `开始分析`。
6. 查看路径还原、案例分析、报告草稿和异常证据。
7. 管理员进行人工确认或驳回。

Dify 接入规范和模板位于：

```text

docs/latest/traffic-audit/dify-workflows.md
```

一期推荐 Dify 只承担：

- 报告草稿生成
- 政策依据摘要
- 绿通辅助核查草稿

Dify 输出始终是草稿，不改变案件事实和人工复核结论。

## 知识边界

为了保证稽核结果可追溯、可复核：

- 原始通行流水不写入知识库。
- 原始通行流水不进入知识图谱。
- OCR、扫描件和复杂版式文档不自动抽取正式图谱关系。
- 专业三元组应由人工或业务系统提供，并经过审核。
- ColPali/视觉 RAG 仅作为后续 PoC，不进入一期主流程。

## 验收标准

- 相同数据重复分析结果一致。
- 每个异常结论都有源记录和规则依据。
- 证据项可定位到源文件、行号和原始记录 ID。
- 业务事实由确定性程序生成。
- LLM/Dify 不自动定案。
- 未经人工确认不得标记为正式结论。

## 项目特点

- 将通用知识库/图谱平台能力收敛到交通稽核业务场景。
- 使用确定性规则承担事实判断，降低 LLM 幻觉对业务结论的影响。
- 保留知识库、图谱、问答和 Dify 的扩展空间，用于依据检索和解释生成。
- 通过独立面板展示路径还原、案例分析和报告生成，避免把业务流程混在工具调用结果中。
- 使用测试数据和 manifest 标注验证规则命中、证据溯源和重复分析一致性。
